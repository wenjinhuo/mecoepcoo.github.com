一、什么是Github？

1. 初识Git

Git是一个开源的Linux下的分布式版本控制系统，他的作者是Linux系统的开发者Linus Torvalds。现在，Git已经能够在Windows下使用。

什么是版本控制系统呢？

想象一下这样的场景：

有一天你统计了一份客户名单，保存为 客户名单.doc ，你的的同事想要补充一下这个名单，于是你通过QQ把文件发给同事，同事直接修改完之后又发回给你，这时你不知道同事有没有不小心删掉或改错了原本你做的东西，所以你把同事发给你的文件另存为 客户名单2.doc ，然后等有空了再一一检查合并，反复传递之后，你的文件夹里有 客户名单1-6 这么多个文件，当你的领导跟你要最终版本时，你还要一个一个去翻找最新改动的那个。

是不是很麻烦呢？

于是，版本控制系统诞生了，他给你的每个文件都留下一个独一无二的版本号，并按照修改日期排序，还附带了是谁发布了这个文件，写明了备注，这就是Git的雏形。

2. 集中式和分布式

常用的集中式版本控制系统有 CVS 和 SVN ，而Git是分布式版本控制系统。

集中式版本控制系统中的文件存放在中央服务器，当多人同时工作时，每人先各自从中央服务器获得最新的版本，修改完自己的部分之后再上传到中央服务器，这样的管理方式必须联网才能有效完成，如果程序员的网速比较慢，将花费大量的时间在数据传输上。

集中式版本控制系统示意图：

分布式版本控制系统弥补了这个缺陷，他把文件存放在每个人的电脑中，不需要中央服务器，如果A和B都修改了各自的文件，他们只需要把有修改的地方相互发送给对方就可以了，大大提高了文件传输的效率。在实际使用中，分布式版本控制系统一般也使用一个服务器用以所有的修改。

分布式版本控制系统示意图：

3. Github

Github是一个开源代码托管库和在线版本控制系统，开发者可以把自己的开源代码免费托管在Github上，并使用Github提供的git功能，同时，Github也允许开发者免费发布一个静态站点。

托管在Github上的最热门的开源项目有：

  • Bootstrap
  • jquery
  • node.js
  • rails
  • jekyll
  • Hexo
  • ……

二、使用Github For Windows

以下同步介绍Github桌面版的使用方法。

1. 安装git

访问https://desktop.github.com/

点击蓝色的“Download Github Desktop”按钮下载github桌面版。安装的过程可能特别缓慢，如果没有报错提示，那就等他自己安装完成，不要关闭窗口。

2. 注册帐户

登录github官网https://github.com/

按提示注册帐户：

3. 新建仓库

安装完github desktop后会得到如下两个图标：

打开Github（左边），填写用户名和密码登录：

选择左上角的加号，在“creat（创建）”中填写仓库名和路径，新建仓库：

点击左上角的“public（发布）”，把新建的仓库上传到github：

4. 提交代码

将要提交的程序存放到本地仓库对应的目录下，随后会在github桌面版中看到有修改的部分，写好总结（summary）和描述（description），点击commit后，点击右上角的sync（同步）来提交：

5. 创建分支

在分支上工作可以让你的编辑不影响到其他分支的代码，适用于多人同时工作的场景。点击分支图标，填写新分支名称后确认，然后点击右上角的sync来提交至github云端：
